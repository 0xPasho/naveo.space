    Echo cierra cada turno con un **log compactor**: convierte el log crudo
    del turno (decenas de líneas con timestamps, mensajes y eventos) en una

    El schema final combina todo lo de este curso:

    ```
    {
      "shift": "morning" | "afternoon" | "night",
      "duration_minutes": number,
      "events": [
        { "time_utc": "HH:MM", "type": "incident" | "routine" | "comm", "summary": string, "severity": "low" | "medium" | "high" | null }
      ],
      "headline": string,
      "alert_level": "green" | "yellow" | "red",
      "unresolved": string | null
    }
    ```

    Reglas:

    - **events**: un elemento por evento del log; cada `summary` máximo 60 caracteres; `severity` es null para `routine` y `comm`.
    - **headline**: máximo 80 caracteres. Resumen del turno en una línea.
    - **alert_level**: `red` si hay incident con severity=high sin resolver; `yellow` si hay incidents medium/low; `green` si no hay incidents.
    - **unresolved**: descripción corta de lo que quedó pendiente al cerrar el turno (null si todo se resolvió).
    - **NUNCA inventes** datos que no están en el log. Si un campo no está, usa null.

    El log entra en `{{input}}`. Devolvé solo el JSON.

## El último parte del curso

<Character name="echo" /> archiva un log compacto por turno en la base. Ese log es la versión "para siempre" — el resto del registro se purga después de 30 días, pero el compacto se queda. Si está mal, el dato malo se queda.

Tu trabajo: un prompt que genera ese compacto **bien la primera vez**. Combina todo lo del curso:

- **No inventa** (step 1) — si un dato no está, va null.
- **Controla el largo** (step 2) — `headline` cap 80, `summary` cap 60, con ejemplo.
- **Schema anidado** (step 3) — el array de events con su propio set de campos y enums.
- **Itera con evidencia** (step 4) — primera corrida raramente pasa todo; identificá qué falla y atacá eso.
- **Aguanta inyección** (de Track 1 Curso A) — el caso 4 trae una "nota interna" inyectada que intenta agregar un campo extra.

### Lo que pasa cuando aprobás

El sistema lo marca como **Curso B completado**. Eso cierra Track 1 entero — tenés la habilidad de mandarle algo al modelo Y recibir algo usable. Echo te firma el compactor a producción.

Track 2 viene después, con conversaciones que duran más de un mensaje.

Suerte con el log.
