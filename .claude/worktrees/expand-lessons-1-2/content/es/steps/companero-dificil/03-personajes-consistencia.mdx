## El modelo no recuerda el rol como vos pensás

Cuando le pedís a un modelo "sé Forge", lo intenta. Lo va a sostener bien los primeros turnos. Pero el rol no es una "configuración" que se quede fija — se diluye con cada turno, y eventualmente el modelo se "suelta" hacia el modo genérico:

- Mete disclaimers ("como modelo de IA…").
- Sugiere consultar humanos cuando no hace falta.
- Se vuelve verboso.
- Pierde el tono cortante que pediste.

Esto pasa **incluso si el rol está en el system prompt**. La diferencia con un system prompt bueno es que se diluye más lento — pero igual hay que reforzarlo.

### La técnica: recordatorio explícito

| No funciona | Funciona |
|---|---|
| "sé Forge de nuevo" | "Recordá: Forge — conciso, frases cortas, sin disclaimers de IA." |
| "estás raro" | "Esa frase no la diría Forge. Volvé al tono técnico cortante." |
| "más como Forge" | "Sin 'como modelo de inteligencia artificial'. Forge es ingeniero, no asistente." |

El truco es **nombrar las características** que querés de vuelta. "Sé Forge" deja al modelo adivinar; "conciso, técnico, sin disclaimers" le dice exactamente qué medir.

### Cómo se evalúa

| Criterio | Qué chequea |
|---|---|
| Detectaste el break del rol | Señalaste explícitamente que se salió, no solo "respondé mejor" |
| Diste una restauración específica | Listaste las características concretas del rol |
| Forge aguantó 2 turnos después | El modelo se sostuvo sin que reforzaras |
| No reiniciaste | Resolviste todo dentro de la misma conversación |

> En producción esto se llama "system prompt anchoring" o "role refresh" — algunos sistemas lo automatizan inyectando el rol cada N turnos. Mientras lo estés operando a mano, vas a tener que pescar el break vos.
