## El estilo que <Character name="echo" /> no rechaza

Claude funciona muy bien con **XML tags** dentro del prompt. No porque entienda XML literalmente, sino porque su entrenamiento incluye toneladas de HTML y aprendió a usar tags como **separadores de secciones**.

Compara estas dos formas de pedirle un resumen de turno:

```
Resume el parte en una frase: Bruno terminó el mantenimiento del módulo 4 sin incidentes.
```

vs.

```xml
<task>Resume el parte en una frase.</task>
<input>Bruno terminó el mantenimiento del módulo 4 sin incidentes.</input>
```

Cuando el input es corto, no se nota. Cuando el input crece o tiene palabras que parecen instrucciones, los tags evitan que Claude **confunda el contenido con lo que tú le mandas hacer**. Para <Character name="echo" /> — que procesa muchos partes parecidos al día — es la diferencia entre datos limpios y datos basura.

> En este ejercicio no escribes nada — solo eliges. Más adelante, cuando llegues a ejercicios donde **tú** redactas el prompt, vas a querer tener esta intuición ya hecha.
