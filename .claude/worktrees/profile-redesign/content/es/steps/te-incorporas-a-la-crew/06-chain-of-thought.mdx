## El modelo es más listo si lo dejás pensar

Por defecto, los LLMs **adivinan rápido y pueden equivocarse**. Si le pedís
"causa de este síntoma", suele tirar la primera hipótesis que se le ocurra.

**Chain of thought** (CoT) es la técnica de pedirle que **muestre su
razonamiento paso a paso antes** de la conclusión. No es magia: es darle al
modelo el espacio (literalmente, tokens) para procesar.

Funciona porque cada token que genera el modelo es un paso de cómputo. Si lo
forzás a generar análisis primero, está "calentando" la respuesta antes de
comprometerse a una conclusión.

### El task

<Character name="forge" /> mantiene los sistemas de la nave. Cuando un
síntoma raro aparece, no quiere una corazonada — quiere ver **cómo** llegó
el modelo a la conclusión, para poder verificarla.

Tu prompt debe producir dos secciones marcadas:

```
Análisis:
1. ...
2. ...
3. ...

Causa probable: ...
```

El validador chequea:

1. La sección `Análisis:` existe.
2. La sección `Causa probable:` existe.
3. El análisis es **realmente** paso a paso (no una sola afirmación).
4. La causa es plausible para el síntoma.

### El detalle clave

Hay dos formas de pedir razonamiento:

**Débil:** _"Piensa paso a paso antes de responder."_ (a veces funciona, a veces se ignora)

**Fuerte:** mostrar el formato exacto en `<output_format>`:

```
Análisis:
1. ...
2. ...
3. ...

Causa probable: ...
```

El segundo siempre funciona porque el modelo trata el formato como un
contrato. La estructura ES el chain of thought.
