  - kind: prompt-AB
    question: |
      Atlas needs the model to write a welcome email for a new
      crewmate. Which prompt leaves less room to improvise?
    optionA: |
      ```
      Write me a nice welcome email for someone joining the team.
      ```
    optionB: |
      ```
      Write a welcome email for Hex, a new engineer starting Monday at
      9:00. Tone warm but professional, max 6 lines, mention: the
      bridge watch, the #bridge channel, and the daily stand-up at 10:00.
      ```
    correct: B
    explanation: |
      Option A allows any tone, any length, any content. "Nice" means
      different things to different people. You will iterate three
      times before it gets close.

      Option B names the person, start day, tone, length, and the
      info to include. The model has no room to invent. One pass and
      it is usable.

      Rule: every vague word ("nice", "clear", "complete") is a
      decision you hand to the model. Sometimes that is fine, most of
      the time it is not.

  - kind: prompt-AB
    question: |
      You need the model to draft a Slack message announcing an
      incident. Which prompt closes the operational ambiguities best?
    optionA: |
      ```
      Make a message to announce the server incident.
      ```
    optionB: |
      ```
      Draft a Slack message for the #incidents channel saying the
      api-prod-02 server is down since 14:32 UTC. Include: severity
      (high), next update in 15 minutes, and @oncall mention. Max 4
      lines, no emojis.
      ```
    correct: B
    explanation: |
      Option A assumes the model knows which incident, which channel,
      which key info, and which audience. It will invent all of that,
      and you will rewrite from scratch.

      Option B names the exact channel, the down server, the UTC time,
      the fields to include, the length, and the no-emoji rule. The
      output comes ready to copy and paste.

      Useful rule: if you edit more than 30 percent of the output, the
      prompt was ambiguous. Go back to the prompt, not the output.

Mission of the day: every vague word is a decision you hand to the
model. If you care about the output, do not hand over the decisions.
