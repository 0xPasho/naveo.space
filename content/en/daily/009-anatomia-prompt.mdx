        label: "Role (who the assistant is)"

  - kind: step-order-dnd
    task: |
      Order the prompt pieces in the order the model should read them,
      top to bottom. The idea is to build context before handing over
      the task.
    steps:
      - id: s1
        label: "Role of the assistant."
        detail: "Who it is and who it is talking to. Without this, the mode is generic."
      - id: s2
        label: "Domain context."
        detail: "Stable data, formats, and IDs that repeat across requests."
      - id: s3
        label: "Concrete task."
        detail: "What this particular request needs to do."
      - id: s4
        label: "Output format."
        detail: "How you want the result so you can consume it downstream."
  - kind: prompt-AB
    question: |
      Vega shows you two versions of the same request. Which one
      separates the prompt pieces better and gives the model a clearer
      shot?
    optionA: |
      ```
      Summarize today's shift report in short bullets, you are an
      experienced analyst and incidents look like INC-XXXX with
      severity low to critical, at most 3 bullets.
      ```
    optionB: |
      ```
      Role: you are a shift analyst with experience.
      Context: incidents IDs INC-XXXX, severity low/med/high/critical.
      Task: summarize today's shift report.
      Format: 3 bullets max, under 15 words each.
      ```
    correct: B
    explanation: |
      Option A packs everything into one sentence. The model has to
      parse what is role, what is context, and what is the actual task.
      The information is there, but tangled.

      Option B separates with visible labels. The model reads "Role",
      "Context", "Task", "Format" and builds the mode step by step. If
      later you want to change only the format, you know exactly which
      line to touch.

      Rule: the prompt pieces are the same. What changes is how
      visible you make them. More visible, easier to iterate.

Mission of the day: if you can name the pieces, you can move them.
Role, context, task, format. Four drawers, not one big drawer.
