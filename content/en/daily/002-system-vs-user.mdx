  - kind: prompt-AB
    question: |
      You want the assistant to sign every response with the exact line
      "Crew on duty." at the end. It must happen in every response, every
      day. Where do you put it?
    optionA: |
      ```
      [system]
      You are the shift coordinator. Always close every response with
      the exact line: "Crew on duty."
      ```
    optionB: |
      ```
      [user]
      Summarize today's report. Remember to close with "Crew on duty."
      ```
    correct: A
    explanation: |
      If the behavior repeats on every interaction, it belongs in the
      system. That is the mental check: "does this apply to every request
      I will ever send, or just this one?"

      "Close with 'Crew on duty.'" applies to every one. If you put it in
      the user message, you have to copy it into every prompt and you will
      forget eventually. The assistant stops doing it and it looks like a
      bug, but really the instruction just went missing.

      In the system it stays persistent: the model applies it to the first
      response, to the fourth, and to the one you send tomorrow without
      resetting anything. That is why the system is for rules that do not
      change.

    parts:

  - kind: prompt-AB
    question: |
      Forge needs to summarize the specific report from today's 2:00 PM
      shift. It is a one-off task: today is that report, tomorrow will be
      another one. Where does the instruction go?
    optionA: |
      ```
      [system]
      Summarize today's 2:00 PM to 10:00 PM shift report in 3 bullets.
      ```
    optionB: |
      ```
      [user]
      Summarize today's 2:00 PM to 10:00 PM shift report in 3 bullets.
      ```
    correct: B
    explanation: |
      Mirror rule from the previous scene: if the instruction changes
      tomorrow, it belongs in user. Today's report is not tomorrow's,
      and the assistant has to be ready for you to ask different things
      every day without touching the system.
      
      If you put today's report in the system, tomorrow you have two bad
      options: change the system (breaks the session, loses context) or
      ask again in user (the instruction shows up twice and the model
      gets confused about which report is the real one).
      
      System = stable rules. User = the request of the moment. If you
      hesitate, ask yourself "will this change tomorrow?". If the answer
      is yes, it goes in user.
