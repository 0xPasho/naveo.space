  Before learning tricks, it helps to know what a prompt is. It is the instruction you give the model. Not magic, not guessing.

  - kind: prompt-AB
    question: |
      These two prompts ask for the same thing. Which one has a better
      shot at returning something useful?
    optionA: |
      ```
      Give me 3 name ideas for my support crew.
      ```
    optionB: |
      ```
      ideas?
      ```
    correct: A
    explanation: |
      The model does not read your mind. If you throw "ideas?" at it
      without saying ideas about what, it will make something up or ask
      you to clarify.

      Option A tells it three things: how many ideas (3), about what
      (naming your crew), and the context (support). That alone is a
      decent floor.

      Basic rule: if you could not fulfill the request just by reading
      it, the model cannot either. Make sure the prompt stands on its
      own.
  - kind: prompt-anatomy
    task: |
      Vega builds a simple prompt piece by piece. Label each line with
      the category it belongs to.
    parts:
      - id: who
        label: "Who I am"
        text: "I lead a small customer support crew."
      - id: what
        label: "What I want"
        text: "I need three name ideas for the team's internal channel."
      - id: how

      - id: context
        label: "Context"
        text: "The team is informal and works remotely from Latam."
  - kind: prompt-AB
    question: |
      You need to ask the model for three different things about the
      same text. Which version makes the model's job easier?
    optionA: |
      ```
      Summarize this text and translate it to English and give me 3
      title ideas and return it all together.
      ```
    optionB: |
      ```
      I have a text. I need three things:
      1) A summary in 2 bullets.
      2) An English translation.
      3) Three title ideas.
      ```
    correct: B
    explanation: |
      Models do not read your mind, but they do read structure. When
      you write "1, 2, 3" the model sees three deliverables and keeps
      them separate.

      In A everything is glued with "and". The model can easily skip
      one of the asks or smash them into a single block that is hard to
      use afterwards.

      Rule: if your prompt asks for more than one thing, number it. If
      it is a single thing, one line is enough.

A prompt is an instruction. If you could not understand it read cold,
the model cannot either. Tell it who you are, what you want, and how
you want it.
