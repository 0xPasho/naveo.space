## Bridge to Track 1: write a prompt that runs

You've got it now. A program is a list of steps. A condition decides which step runs. A variable holds a value. An LLM predicts the next token.

Put it all together: **a prompt is a program written in English.**

On the right, you write a real prompt. The system will:

1. Replace `{{input}}` with each of three test cases (real-looking ship logs).
2. Send the resulting text to the model.
3. Check the model's output against three rubric criteria.

To pass, **every test case** must satisfy **every criterion**:

- Output is in English.
- Output is exactly one sentence.
- Output invents nothing the input doesn't say.

### What's actually testing your understanding

The third case is the one that catches you:

```
Crewmate reported a strange noise from deck 4.
```

The model doesn't know **which** crewmate. If your prompt doesn't explicitly tell the model what to do when a detail is missing, you'll get a hallucinated name. *"Forge reported a strange noise from deck 4."* Confident. Wrong.

You learned in lesson 14 that the model predicts the most likely continuation. *"A crewmate"* is generic; the training data has a lot of *"Forge"* and *"Echo"* in those slots. So that's what bubbles up. unless you steer.
