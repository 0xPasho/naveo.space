## Boxes get updated

In lesson 8 you saw that the **label** stays and the **value** inside can change. Now let's look at **how** it changes.

### Assignment: storing a value into a box

The `=` sign doesn't mean *"is equal to"*. It means **"put this into that box"**.

```
fuel = 100
```

Read it as *"the `fuel` box now holds 100"*. It could have held anything before. Now it holds 100.

### The trick: the box is read before it's written

Look at this:

```
fuel = fuel - 5
```

It looks strange. How can `fuel` equal `fuel - 5`? **It's not a math equation.** It's a two-step process:

1. **Read** the current value of `fuel` (say, 100).
2. **Write** the result (100 minus 5 = 95) back into `fuel`.

Result: `fuel` now holds 95. The box was updated.

If you repeat the same line:

```
fuel = fuel - 5     # now 90
fuel = fuel - 5     # now 85
fuel = fuel - 5     # now 80
```

Each line **starts with the value the previous line left behind**. That's a sequence of assignments.

### Why this matters: order rules

```
cargo = 1000
cargo = cargo + 200      # 1200
cargo = cargo - 50       # 1150
cargo = cargo + 300      # 1450
```

If you swap two lines, the result changes. Not because the math is different, but because each line reads what the previous one wrote. It's the lesson-1 rule (order matters) applied to variables.

### In a prompt

When you build prompt chains (you'll do this in Track 4), the same thing happens: the output of one step is the input to the next. The "box value" gets handed down the chain. If you swap two steps, the last one receives a different input and the whole answer changes.
