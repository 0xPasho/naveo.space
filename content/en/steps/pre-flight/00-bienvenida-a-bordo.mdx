## <Character name="vega" /> meets you at the airlock

The ship runs on instructions. Not screams, not vibes. **instructions** the crew can read top-to-bottom and act on. If you can give a clean instruction, the crew can do anything for you.

This pre-flight isn't a programming bootcamp. It's a **language lesson**: the minimum grammar the ship understands. Twenty-one short rituals, grouped into six units. Then Track 1.

What you'll pick up, unit by unit:

- **Steps.** Why order is the difference between *we landed* and *we crashed*, and how to find bugs that don't announce themselves.
- **Decisions.** How the crew picks between two (or three, or five) paths. Operators `<`, `>`, `==`, `AND`, `OR`, and the boundary trap.
- **Variables and data.** The labeled boxes the ship moves around. Value types, lists, and why boxes change over time.
- **Functions.** Protocols with a name. The idea of input and output, which is exactly how you'll think about the LLM.
- **How the model thinks.** What's actually happening when you "talk to it". Tokens, context, sampling, hallucinations.
- **Your first prompt.** Tie it all together and write one that passes a real rubric.

> If you already write code for a living, the catalog has a skip button for this track. <Character name="vega" /> won't be offended.

When you're ready, advance. <Character name="vega" /> is waiting at the first console.
