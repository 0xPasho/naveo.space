## Repeating the same thing, many times

Sometimes you don't need to write the same step six times. You write it **once**, and tell the ship: *do this for each crewmate*.

That's a **loop**. In code it looks like:

```
FOR EACH crewmate IN crew:
  check suit oxygen
```

That snippet "expands" at runtime. If the crew is 4 people, it runs 4 times. If it's 12, it runs 12 times. **You don't count.** The loop counts.

### Two versions of the same job

Without a loop:

```
Check Vega's suit oxygen
Check Atlas's suit oxygen
Check Echo's suit oxygen
Check Forge's suit oxygen
```

With a loop:

```
FOR EACH crewmate IN crew:
  Check suit oxygen
```

The second one isn't just shorter. It's **correct when the crew changes**. If someone new joins tomorrow, the loop includes them automatically. The no-loop version ignores newcomers until you remember to add a line.

### How to spot a loop

Look at whether the action's subject **changes between repetitions**:

- *Check suit oxygen*: each suit belongs to a different person. **Loop.**
- *Fire the main engines*: the engines are the same for everyone. **Do once.**

If the subject is always the same, you don't need a loop. If the subject varies across runs, you do.

> When you send the model a prompt with a list inside (for example *"for each bio, extract the email"*), you're using this exact shape. The loop is inside the prompt and runs once per bio.
