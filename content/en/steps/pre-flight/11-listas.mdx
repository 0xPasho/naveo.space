## One box with many things inside

So far boxes held a single value: a number, a text, a true/false. But the ship needs to hold **collections** of things. The crew has four members, not one.

That's what a **list** is for: a box with several values inside, in order.

```
crew = ["Vega", "Atlas", "Echo", "Forge"]
```

That box holds four texts in a sequence. The list preserves **order**: the position matters, it's not an unordered set.

### Accessing by position

To pull a value out of the list you use a number in brackets, the **position**:

```
crew[0]  → "Vega"      ← the first one
crew[1]  → "Atlas"
crew[2]  → "Echo"
crew[3]  → "Forge"     ← the fourth and last
```

### The surprising detail: it starts at 0

In almost every programming language, **the first position is 0, not 1**. The fourth is 3, not 4. The last is always `count - 1`.

For a crew of 4:

- `crew[0]` is the first (Vega)
- `crew[3]` is the last (Forge)
- `crew[4]` **doesn't exist** (error or null, depending on the language)

It's counterintuitive at first, but there's a historical reason and you'll internalize it quickly.

### Typical operations

Lists have their own vocabulary:

- `len(crew)` counts how many there are (4)
- `crew[0]` pulls one out by position ("Vega")
- `crew.append("Orbit")` adds to the end: `["Vega", "Atlas", "Echo", "Forge", "Orbit"]`
- `crew.remove("Vega")` takes one out by name: `["Atlas", "Echo", "Forge"]`

### Lists + loops: the most common combination

Lists pair particularly well with loops (lesson 3):

```
FOR EACH crewmate IN crew:
  check suit oxygen
```

The loop visits each position in the list, one at a time. Whether the crew is 4 or 40, the code is the same. The loop adjusts itself.

> Almost every structured input to an LLM is a list: list of messages in a conversation, list of bios to process, list of available tools. The "many items in order" idea is **everywhere** once you move past trivial prompts.
