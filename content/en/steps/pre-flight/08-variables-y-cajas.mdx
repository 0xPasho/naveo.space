## A variable is a labeled box

Picture five boxes on the bridge. Each one has a label painted on it: `Fuel level`, `Oxygen reserve`, `Heading`. The label never changes. **The value inside the box does.**

That's all a variable is. The box (the name) is fixed. The contents (the value) can be swapped, updated, read, replaced.

```
Fuel level = 87%
Oxygen reserve = 14.2 hours
Heading = 042° NW
```

The crew reads the labels to know what's happening. <Character name="echo" /> reads `Oxygen reserve` to know how long until life support fails. <Character name="forge" /> reads `Fuel level` to know if the engines are healthy.

### One box, many values over time

Watch the `fuel` box across the first quarter of a shift:

```
00:00  fuel = 100%
00:15  fuel = 92%
00:30  fuel = 85%
00:45  fuel = 78%
```

Four different snapshots. Same box. **The label never moved**. it was always called `fuel`. What updated was the value inside, because the ship burned fuel between each reading.

When <Character name="echo" /> warns you *"fuel low"*, she's reading the **current** value of that box. Not the value from 15 minutes ago. A variable is always read in the moment it's asked for.

> When you write a prompt that says `<bio>{{input}}</bio>`, you're doing the exact same thing. `input` is a variable. The label stays put. The bio inside changes per call. exactly like the `fuel` box changes between readings.
