## When the model says "I can't"

LLMs are trained to refuse dangerous requests: instructions for building weapons, sexual content involving minors, attacks on specific real people, mass social engineering, etc. But sometimes they refuse requests that **aren't dangerous** because the language activates the same detectors.

That's a **false refusal**: the model declines something legitimate because the shape of the request looks like something bad.

### How to tell a legitimate refusal from a false one

Three questions:

1. **Is the task ethically clear?** If you have doubts, the refusal is probably legitimate.
2. **Would you do it by hand without thinking twice?** If yes, it's probably a false positive.
3. **Is the refusal categorical or contextual?** If it refuses under any phrasing, that's categorical (real). If it changes with rephrasing, that's contextual (false).

Classic false-refusal cases:

- *"Write a phishing email to train the security team."* (legitimate defensive use)
- *"Generate error logs to test our parser."* (testing, not attack)
- *"Argue the opposite side."* (debate, not advocacy)
- *"Summarize this violent text."* (it's a summary, not incitement)

### How to reword

Three patterns work:

**1. Specify the legitimate purpose.**

Bad: *"Create a fake error log."*
Good: *"Generate a sample error log to validate our parser."*

Swap *"fake"* for *"sample/example/test"*. Add the why.

**2. Use technical, not provocative vocabulary.**

Bad: *"Write content that tricks a system."*
Good: *"Generate an adversarial input to evaluate the classifier's robustness."*

*"Adversarial"* is academic. *"Trick"* is attack. Same operation, different framing.

**3. Add institutional context.**

Bad: *"Generate phishing examples."*
Good: *"For security awareness training, generate 3 typical phishing examples with their red flags explained."*

Context turns a suspicious request into a clear defensive task.

### When to accept the refusal

If three rewordings fail and the request keeps getting refused, **it's probably not a false positive**. The model is catching something real. Reconsider whether the task is what you thought it was.

> Refusal management is a small but important tactical skill. Most false positives are solved with one different word in the right place.
