## When the prompt sounds clear but isn't

There's a failure category that's hard to spot because the prompt **seems** clear. You read it and you understand what you're asking. The problem is that **another reasonable person** could read it and understand something different.

That's ambiguity. And the model, when it hits ambiguity, **picks an interpretation each time**, not always the same one. Result: the same prompt on the same input can give different responses on different runs.

### The most common ambiguity traps

**1. Relative time references.**

- *"The most recent reports"*: from the last day? the last N? since when is "recent"?
- *"Last month's expenses"*: last calendar month? the last 30 days?
- *"What's new"*: newer than what?

**Fix:** convert "recent/new/last" into a concrete numerical criterion. *"The 5 with the highest timestamp."*

**2. Pronouns and references.**

- *"Summarize it and return it as JSON."*: the summary in JSON, or JSON containing the summary?
- *"Process it and save it."*: the original, the result, both?

**Fix:** repeat the noun. *"Summarize it. Return the summary as JSON."*

**3. Vague quantifiers.**

- *"Summarize the important points."*: how many? what makes something "important"?
- *"Mention a few examples."*: 2? 5? 10?
- *"Be brief."*: one line? one paragraph?

**Fix:** convert "some/several/few/brief" into exact numbers or structures.

**4. Inclusion/exclusion.**

- *"Process the urgent reports."*: only urgent? urgent + high? what do I do with the rest?
- *"Return the valid emails."*: only the valid ones? do I ignore the invalid ones? report them?

**Fix:** say what to do in each case, not just the main case.

### How to spot ambiguity before it bites

Before you send your prompt, read it and ask yourself:

1. Is there a quantity word without a number? (several, some, few, recent...)
2. Is there a time reference without a date? (last, recent, new...)
3. Is there a pronoun that points to more than one thing?
4. Is there an instruction that omits what to do with non-main cases?

Each "yes" is an open door to different interpretations.

> Heuristic: if after reading your prompt you could write 2 equally valid outputs, so can the model. It'll pick one at random. Close the ambiguity before sending.
