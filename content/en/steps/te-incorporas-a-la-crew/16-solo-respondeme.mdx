## The model is a big dog: friendly but chatty

<Character name="forge" /> has a simple rule in her pipeline: the model's output goes **straight** into `JSON.parse(...)`. No cleanup, no regex pre-pass, no "let's strip the backticks it adds sometimes".

That means outputs like these blow up the system:

- **Output with preamble:** the model first writes "Here's the JSON you asked for:" then the JSON on the line below. `JSON.parse` sees the first line and dies.
- **Output wrapped in a markdown block:** the model adds three backticks plus `json` before the JSON, and three backticks after. `JSON.parse` reads the first backtick and dies.
- **Output with conversational closing:** the JSON comes in the middle, with text before ("Sure, based on the report:") and after ("Need anything else?"). `JSON.parse` dies on the first non-JSON character.

All three have the correct JSON inside. but the parser only sees the first line and falls over. Your job: **force the model to shut up**.

### Three techniques that work

1. **Say it explicitly in the prompt.** "Respond ONLY with the JSON, no text before or after." Sounds obvious, but most prompts don't say it and the model defaults to being helpful.

2. **Show the first character.** A line like "Your output starts with open-brace and ends with close-brace. Nothing else." binds the model to a syntactic constraint, not a pragmatic one. Works much better.

3. **Forbid code fences specifically.** Claude tends to wrap JSON in markdown blocks. If your pipeline doesn't expect them, spell it out: "No code fences. No triple backtick. Just the raw JSON."

### How it's evaluated

6 deterministic criteria + 1 judge:

1. Output starts with an open-brace.
2. Output ends with a close-brace.
3. Contains no triple backticks anywhere.
4. Parseable JSON.
5. Has the 3 correct keys.
6. Values are correct for each report.

All 3 cases must pass **every** criterion. The judge is cheap. if you fail any deterministic check, it doesn't run.

> Tip: pass the deterministic criteria first. Once all of those are green, the judge almost never fails. the model is already being orderly.
