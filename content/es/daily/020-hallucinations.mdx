    question: |
      Echo le pidió al modelo "dame el changelog de la version 3.2 de
      mi-libreria interna". El modelo nunca vio esa libreria, pero
      respondió con un changelog detallado y plausible. ¿Cuál es el
      problema de raíz?
    optionA: |
      ```
      El modelo no tiene acceso a internet. Habria que activarle
      browsing o conectar un retrieval para que pueda leer el repo
      real.
      ```
    optionB: |
      ```
      El prompt no le dio al modelo permiso para decir "no se". Sin
      esa salida, el modelo elige siempre la opcion mas probable, y
      esa opcion suele ser un changelog plausible aunque sea ficcion.
      ```
    correct: B
    explanation: |
      La A apunta a una solución (retrieval), no a la causa. La
      libreria interna del usuario no está en internet, browsing no
      la encuentra. El problema es anterior.

      La B nombra la causa real: cuando el modelo no tiene
      información, su default no es callarse, es completar con la
      respuesta más probable estadísticamente. "Changelog v3.2" tiene
      una forma muy estable en sus datos de entrenamiento (versiones,
      bullets, "Added/Fixed/Changed"), entonces produce algo que
      matchea esa forma, aunque el contenido sea inventado.

      Mitigación: explícitamente decir en el prompt "si no tenes esta
      info, responde 'no la tengo, necesito el README o el changelog
      real'". Le das una salida, y el modelo la prefiere a inventar.

  - kind: prompt-AB
    question: |
      Vega arma un prompt que va a consumir datos potencialmente
      desconocidos para el modelo. Tiene dos formas de pedir citas.
      ¿Cuál minimiza más la chance de alucinación?
    optionA: |
      ```
      Después de cada respuesta, agregá las fuentes que usaste.
      Idealmente con links.
      ```
    optionB: |
      ```
      Antes de afirmar cada hecho, citá entre comillas la fuente
      exacta de donde sale (linea del documento o link). Si no
      podes citar, no podes afirmar: responde "no tengo fuente para
      esto". El check de citas es PARTE de la respuesta, no un
      agregado al final.
      ```
    correct: B
    explanation: |
      El A pide la cita despues, como apéndice. El problema: el
      modelo ya completó la afirmación, ya está la oración inventada
      arriba. Después rellena el campo "fuentes" con algo que suene
      bien, y vos terminás con texto plausible + citas inventadas.

      El B reordena el flujo: la cita es prerequisito de la
      afirmación. Si el modelo no puede citar, no puede afirmar. Eso
      cambia el behavior porque la salida "no tengo fuente" queda
      explícitamente disponible y autorizada. El modelo prefiere
      decir "no sé" cuando esa salida es legítima.

      Regla: pedir citas como add-on es decorativo. Pedirlas como
      condición de la afirmación es la mitigación real.
---

Misión del día: el modelo alucina cuando le pedís lo que no sabe y no le
das salida. Decile "podés decir no sé", pedí fuentes antes de
afirmaciones, y verificá lo crítico afuera.
