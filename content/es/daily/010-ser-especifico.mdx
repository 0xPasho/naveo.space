  - kind: prompt-AB
    question: |
      Atlas necesita que el modelo le escriba un email de bienvenida
      para un crewmate nuevo. ¿Cuál prompt le deja menos espacio para
      improvisar?
    optionA: |
      ```
      Escribime un email lindo de bienvenida para alguien que se suma.
      ```
    optionB: |
      ```
      Escribime un email de bienvenida para Hex, ingeniera nueva que
      arranca el lunes 9:00. Tono cálido pero profesional, máximo 6
      líneas, mencioná: la guardia, el canal #bridge, y el horario
      de la stand-up diaria 10:00.
      ```
    correct: B
    explanation: |
      El A admite cualquier tono, cualquier largo, cualquier contenido.
      "Lindo" significa cosas distintas para cada persona. Vas a tener
      que iterar tres veces hasta que se acerque.

      El B nombra a quién, cuándo arranca, qué tono, qué largo, y qué
      info incluir. El modelo no tiene espacio para inventar. Una
      sola pasada y queda usable.

      Regla: cada palabra vaga ("lindo", "claro", "completo") es una
      decisión que le estás cediendo al modelo. A veces conviene; la
      mayoría no.

  - kind: prompt-AB
    question: |
      Tenés que pedirle al modelo que genere un mensaje de Slack para
      avisar de un incidente. ¿Cuál de los dos prompts cierra mejor las
      ambigüedades operacionales?
    optionA: |
      ```
      Hacé un mensaje para avisar del incidente del servidor.
      ```
    optionB: |
      ```
      Generá un mensaje de Slack para el canal #incidents avisando que
      el servidor api-prod-02 está caído desde las 14:32 UTC. Incluí:
      severidad (high), próximo update en 15 minutos, y mención a
      @oncall. Máximo 4 líneas, sin emojis.
      ```
    correct: B
    explanation: |
      El A asume que el modelo sabe a qué incidente, en qué canal, con
      qué información clave, y para qué audiencia. Va a inventar todo
      eso y vos vas a tener que reescribir desde cero.

      El B nombra el canal exacto, el server caído, la hora UTC, los
      campos a incluir, el largo, y la prohibición de emojis. El
      output sale listo para copiar y pegar.

      Una regla útil: si después del output tenés que editar más del
      30 por ciento, tu prompt era ambiguo. Volvé al prompt, no al
      output.

Misión del día: cada palabra vaga es una decisión que le cedés al
modelo. Si te importa el output, no le cedas las decisiones.
