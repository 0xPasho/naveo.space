  - kind: prompt-AB
    question: |
      Querés que el asistente firme cada respuesta con la frase "Crew on
      duty." al final. Es algo que tiene que pasar en todas las respuestas,
      todos los días. ¿Dónde lo ponés?
    optionA: |
      ```
      [system]
      Sos coordinador de turno. Siempre cerrá cada respuesta con
      la línea exacta: "Crew on duty."
      ```
    optionB: |
      ```
      [user]
      Resumime el reporte de hoy. Acordate de cerrar con "Crew on duty."
      ```
    correct: A
    explanation: |
      Si el comportamiento se repite en cada interacción, va en el system.
      Esa es la regla mental: "¿esto vale para todos los pedidos que voy
      a hacer, o solo para este?"

      "Cerrá con 'Crew on duty.'" vale para todos. Si lo metés en el user,
      tenés que copiarlo en cada mensaje y eventualmente se te olvida en uno.
      El asistente lo deja de hacer y parece bug, pero es porque la
      instrucción se perdió.

      En el system queda persistente: el modelo lo aplica a la primera
      respuesta, a la cuarta, y a la que mandes mañana sin reiniciar nada.
      Por eso el system es para reglas que no cambian.

  - kind: prompt-AB
    question: |
      Forge necesita resumir el reporte específico del turno de las 14:00
      de hoy. Es una tarea puntual: hoy es ese reporte, mañana será otro.
      ¿Dónde va la instrucción?
    optionA: |
      ```
      [system]
      Resumime el reporte del turno de las 14:00 a 22:00 de hoy en 3 bullets.
      ```
    optionB: |
      ```
      [user]
      Resumime el reporte del turno de las 14:00 a 22:00 de hoy en 3 bullets.
      ```
    correct: B
    explanation: |
      La regla espejo de la escena anterior: si la instrucción cambia
      mañana, va en user. El reporte de hoy no es el de mañana, y el
      asistente tiene que estar listo para que vos le pidas cosas
      distintas todos los días sin tocar el system.
      
      Si metés el reporte de hoy en el system, mañana tenés dos opciones
      malas: cambiar el system (rompe la sesión, pierde contexto) o
      pedirlo de nuevo en el user (la instrucción aparece dos veces y
      el modelo se confunde con cuál es el reporte real).
      
      System = reglas estables. User = pedido del momento. Si dudás,
      preguntate "¿esto va a cambiar mañana?". Si la respuesta es sí,
      va en user.

Misión del día: aprender a separar lo estable de lo puntual. El system es
para las reglas. El user es para la tarea.
