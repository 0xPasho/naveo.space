## Hasta acá fueron piezas. Ahora viene el sistema.

Recorriste el camino completo del prompt:

- **Track 1**. un prompt bien estructurado.
- **Track 2**. una conversación bien sostenida.
- **Track 3**. un toolkit bien definido para que el modelo actúe.

Cada pieza por separado resuelve **un escenario acotado**. En la realidad, los problemas son más feos: clasificar 50 tipos de input distintos, encadenar 3-4 pasos donde cada uno toca un dato del anterior, hacer retrieval antes de generar, **medir si todo eso funciona** cuando ya no podés probar mentalmente cada caso.

Para eso necesitás algo distinto: **diseñar el sistema**.

> Un sistema multi-step es **varias piezas LLM (y no-LLM) orquestadas** donde la salida de una alimenta a la siguiente. Diseñar uno bien implica saber dónde partir el problema, cómo enrutar inputs, cuándo parar un loop, y cómo evaluar el resultado completo. no cada pieza suelta.

En este track:

- <Character name="orbit" /> te enseña a **diseñar el flujo**: dónde partir el problema, qué corre en cadena, qué se bifurca, cuándo el loop tiene que cortar.
- <Character name="forge" /> aporta el toolkit (lo que aprendiste en Track 3, ahora orquestado).
- <Character name="echo" /> te enseña a evaluar (cómo saber si tu sistema funciona).
- <Character name="atlas" /> firma el capstone cuando tu sistema completo pasa las evaluaciones.

Es el último track del onboarding. Cuando lo cierres, sos **ingeniero de sistemas con LLMs**, no usuario.
