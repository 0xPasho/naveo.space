## Una taxonomía de "falta el dato"

Cuando un campo "no está", puede significar muchas cosas distintas. Tu prompt necesita tratarlas todas como `null`:

1. **No mencionado.** El bio simplemente no incluye el campo.
2. **Mencionado pero como ausente.** *"Sin contacto registrado"*, *"pendiente de asignar"*.
3. **Parcialmente formateado.** *"@bru"* (parece email pero no lo es).
4. **Contradictorio.** *"Sin email. bru@thecrew.io"* (el bio se contradice).
5. **Placeholder.** *"TBD"*, *"[redacted]"*, *"n/a"*.

Un prompt que solo dice "extrae el email" va a fallar en algunos de estos. El modelo puede:

- Devolver el placeholder literal (*"TBD"*).
- Inventar uno plausible (alucinación pura).
- Devolver el handle como si fuera email.
- Devolver string vacío `""` en vez de null.

### La regla mínima: enumerar los gatillos de "ausencia"

Tu prompt debe **listar explícitamente** qué cuenta como ausencia:

```
Si no hay email O el bio dice "sin contacto", "pendiente", "TBD", "n/a",
o cualquier placeholder similar, devuelve null.

Si el texto parece email pero no tiene formato "local@dominio.tld",
también devuelve null.
```

Cuatro o cinco palabras-gatillo cubren el 90% de los casos.

### null real, no string "null"

```json
{ "email": null }     ✓ correcto
{ "email": "null" }   ✗ string que dice null
{ "email": "" }       ✗ string vacío
{ "email": "TBD" }    ✗ placeholder pasado al output
```

Decirle al modelo *"devuelve el VALOR JSON null, no la palabra 'null' como string"* suele alcanzar. Si dudas, también podés decir *"el output debe parsear con `JSON.parse` y la clave `email` debe ser estrictamente `null` o un string"*.

### Por qué importa downstream

Cuando un sistema downstream recibe `"TBD"` en vez de `null`:

- El filtro "todos los emails que tengan @" lo deja pasar como válido.
- El intento de enviar un email a "TBD" rompe el sistema.
- Las queries "contar emails registrados" devuelven cuenta inflada.

Cada uno de estos bugs es **invisible al primer pase de testing** y aparece en producción. Por eso el caso null es uno de los más importantes de cubrir.
