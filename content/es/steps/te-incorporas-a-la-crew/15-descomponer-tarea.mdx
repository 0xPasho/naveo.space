## Tarea grande, problemas grandes

Cuando le pides al modelo *"resumí, clasificá, y recomendá acción"* en una sola frase, va a:

1. Mezclar las tres cosas en un párrafo.
2. Olvidar una (típicamente la última).
3. Generarlas en distinto orden o formato cada vez.

La solución: **descomponer la tarea en sub-pasos explícitos dentro del prompt**.

### CoT vs descomposición: parecidos pero distintos

| | CoT (lección 14) | Descomposición |
| --- | --- | --- |
| Cuándo | Una tarea que requiere razonamiento profundo | Múltiples tareas en paralelo |
| Estructura | Pasos de pensamiento → conclusión | N outputs etiquetados |
| Ejemplo | "Análisis: ... / Causa: ..." | "SUMMARY: ... / SEVERITY: ... / ACTION: ..." |

CoT es **vertical** (pensar profundo sobre una cosa). Descomposición es **horizontal** (hacer N cosas separadas).

### Cómo descomponer en 3 pasos

**1. Listá las sub-tareas explícitamente** en la instrucción:

```
Esta tarea tiene 3 partes:
1. Resumir el issue (una oración).
2. Clasificar severidad (high, medium, low).
3. Recomendar acción concreta.
```

**2. Mostrá el formato exacto** del output con etiquetas:

```
SUMMARY: ...
SEVERITY: high|medium|low
ACTION: ...
```

**3. Hacé cada parte parseable** desde código.

Si SUMMARY siempre está en su propia línea, podés extraerlo con un regex de una línea. Si está embebido en un párrafo, necesitás otro LLM solo para parsearlo. Eso multiplica costo y errores.

### Por qué funciona

Cada etiqueta es un "compromiso" del modelo. Cuando escribe `SEVERITY:`, las probabilidades del siguiente token colapsan sobre `high|medium|low`. No hay espacio para divagar.

Sin etiquetas, el modelo genera un párrafo donde "severidad" podría aparecer al inicio, al medio, al final, o no aparecer. Con etiquetas, está donde debe estar, en el formato que necesitas.

> El bonus oculto: descomponer también te facilita el **debugging**. Si solo la severidad falla, sabés exactamente dónde mirar. Sin descomposición, "el prompt anda mal" es todo lo que tenés.

A la derecha, tres tickets de mantenimiento. Escribí un prompt que genere los tres campos etiquetados consistentemente.
