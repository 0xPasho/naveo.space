## Prompt engineering = iteración

Casi ningún prompt sale bien al primer intento. La habilidad real no es escribir el prompt perfecto de una. Es **iterar rápido**.

### El loop

```
1. Escribís prompt v1.
2. Corrés contra tus test cases.
3. Mirás qué falla y por qué.
4. Cambiás UNA cosa específica.
5. Volvés a correr.
6. Repetís.
```

La parte que la gente saltea: el **paso 3**. Mirar el output real, no asumir cuál es el problema. Si asumís, terminás cambiando lo que no falla.

### Las 4 categorías de falla

Casi todo bug de prompt cae en una de cuatro categorías. Reconocer cuál te ahorra horas:

**1. Formato.** El contenido es correcto pero rompe el parser (sobra preamble, falta una clave, formato inconsistente).
**Fix:** agregar reglas de formato explícitas, usar `<output_format>`, mostrar el shape exacto del output.

**2. Cobertura.** Algunos casos andan, otros no. El prompt asume que el input es de cierta forma y los casos raros lo rompen.
**Fix:** agregar regla específica para el caso que falla.

**3. Inventos (alucinación).** Devuelve datos que no están en el input.
**Fix:** agregar *"si no aparece, devolvé null"*, explícitamente.

**4. Tono / idioma / longitud.** Lo más fácil. Restricciones específicas resuelven.

### El truco del cambio aislado

Cuando iteras, cambiá **una cosa a la vez**. Tentación: ver 3 problemas, cambiar 3 reglas en el prompt, correr todo. Resultado: si 1 de los 3 fixes empeora otra cosa, no sabés cuál fue.

Disciplina: una iteración = un cambio = una corrida. Aburrido pero mil veces más rápido para llegar al prompt correcto.

### Las 3 preguntas que te haces en cada output fallido

En orden:

1. **¿Cuál criterio falló específicamente?** (Lo dice el feedback.)
2. **¿El output muestra entendimiento parcial o error completo?** (Parcial = ajuste; completo = repensar).
3. **¿Mi prompt menciona explícitamente esta situación?** (Si "no", ya sabés qué agregar.)

> El prompt final que pasa todos los casos rara vez es el más elegante. Es el que sobrevivió a las 4 iteraciones que se necesitaron. No te enamores del v1.

A la derecha, un starter intencionalmente flojo. Corré sin tocar, mirá qué falla, ajustá una cosa. Repetí.
