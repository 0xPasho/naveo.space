## Entrada y salida: el contrato de una función

Una función es una caja negra. Le metés cosas adentro. Te devuelve cosas distintas. Lo que importa entender:

- **Entradas (inputs):** lo que la función necesita para hacer su trabajo. Los parámetros que le pasás cuando la llamás.
- **Salida (output):** lo que la función devuelve al final. El resultado.

```
FUNCION calcularCombustibleRestante(distancia, peso):
  ...cálculos...
  devolver minutos_disponibles
```

- Entradas: `distancia`, `peso`
- Salida: `minutos_disponibles`

### Por qué pensar en entrada/salida importa

Cuando ves una función nueva, antes de leer el cuerpo, mirá la **firma**: nombre + entradas + salida. Eso ya te dice qué hace el 90% de las veces.

Mirá esta:

```
formatearTimestamp(epoch, zona_horaria) → string
```

Sin leer una línea de adentro, ya sabés:

- Necesita un `epoch` (número de segundos) y una `zona_horaria` (texto).
- Te devuelve un `string` formateado.
- No modifica variables globales por afuera, al menos según su firma.

Esa lectura rápida es lo que te permite usar funciones sin tener que entender cada detalle de la implementación.

### Funciones sin salida (o sin entrada)

Algunas no devuelven nada. Solo **hacen** algo:

```
sonarAlarma("crítica", 30)   # entradas: 2. salida: ninguna.
```

Otras no necesitan entradas:

```
horaActual()                 # entradas: ninguna. salida: timestamp.
```

Ambas son funciones legítimas. Pero la mayoría útil tiene **al menos una de cada**.

### El LLM también tiene una firma

Esta es la idea más importante para el resto del curriculum:

```
LLM(prompt) → respuesta
```

El modelo es una función. **Entrada:** un prompt (texto). **Salida:** una respuesta (texto). Punto. Todo lo que aprendiste sobre funciones aplica al modelo:

- Misma entrada exacta puede dar salida distinta (no es 100% determinista, vas a ver por qué en lecciones 17-18).
- Diseñar la entrada (el prompt) bien es la mitad del trabajo. La función no puede inventar lo que no le diste.
- La salida solo es útil si la entrada cubrió todo lo necesario.

> **Esta es la base mental del prompting:** entendés al LLM como una función `texto → texto`. Cuando te frustres porque *"no hace lo que quiero"*, el primer chequeo es siempre la entrada.
