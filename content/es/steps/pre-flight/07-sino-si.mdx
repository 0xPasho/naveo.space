## Cuando hay más de dos opciones

Hasta acá las decisiones eran binarias: `SI` esto, `SINO` aquello. Dos ramas, una decisión.

Pero la mayoría de las decisiones reales tienen **más de dos resultados posibles**. Pensá en el oxígeno:

- 8% → crisis, todos los protocolos de emergencia
- 22% → advertencia seria, prepararse
- 40% → bajo pero manejable, avisar
- 70% → normal, silencio

Para expresar esto vas a encadenar varios `SI`. Se llama **sino-si** (en otros lenguajes: `else if`, `elif`).

### Cómo se escribe

<DecisionChain branches="oxígeno < 10% => alarma crítica; oxígeno < 25% => alarma de advertencia; oxígeno < 50% => notificación informativa" fallback="silencio" />

La nave evalúa **de arriba hacia abajo** y se detiene en el **primero que cumple**. Después salta directo al final.

### La regla clave: gana el primero

Mirá qué pasa con `oxígeno = 8%`:

1. ¿8 < 10? **sí** → corre `alarma crítica` y termina. **El resto no se evalúa.**

Mirá qué pasa con `oxígeno = 22%`:

1. ¿22 < 10? no, sigue.
2. ¿22 < 25? **sí** → corre `alarma de advertencia` y termina.

Y con `oxígeno = 72%`:

1. ¿72 < 10? no.
2. ¿72 < 25? no.
3. ¿72 < 50? no.
4. `SINO` → corre `silencio`.

El último `SINO` (sin condición) es el **resto**: corre solo si ninguna de las anteriores se cumplió. Funciona como red de seguridad.

### El bug clásico: orden invertido

Si invertís el orden de las ramas, todo se rompe sin tirar error.

```
SI oxígeno < 50%:
  notificación informativa
SINO SI oxígeno < 25%:
  alarma de advertencia
SINO SI oxígeno < 10%:
  alarma crítica
```

Con `oxígeno = 8%`, la primera condición (`8 < 50`) **ya es verdadera**. La nave corre *notificación informativa* y se va. La alarma crítica nunca se dispara.

> Regla práctica: en cadenas de `SINO SI` ordenadas por umbral, va primero **el caso más restrictivo** (el rango más chico) y vas relajando hacia abajo.

Vas a ver esta misma cadena en prompts complejos: *"Si el bio tiene email **y** teléfono, devolvé ambos. Si no, si tiene solo email, devolvé el email. Si no, si tiene solo teléfono, devolvé el teléfono. Si no, devolvé null."* Mismo patrón.
