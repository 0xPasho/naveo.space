## Puente al Track 1: escribí un prompt que corra

Ya está. Un programa es una lista de pasos. Una condición decide cuál paso corre. Una variable guarda un valor. Un LLM predice el siguiente token.

Junto todo: **un prompt es un programa escrito en español (o inglés).**

A la derecha escribís un prompt real. El sistema va a:

1. Reemplazar `{{input}}` con cada uno de los tres test cases (logs realistas de la nave).
2. Mandar el texto resultante al modelo.
3. Chequear la salida del modelo contra tres criterios de rúbrica.

Para aprobar, **cada test case** tiene que cumplir **cada criterio**:

- La salida está en español.
- La salida es exactamente una oración.
- La salida no inventa nada que el input no diga.

### Qué te está testeando de verdad

El tercer caso es el que te agarra:

```
Un tripulante reportó un ruido extraño desde la cubierta 4.
```

El modelo no sabe **cuál** tripulante. Si tu prompt no le dice explícitamente qué hacer cuando falta un detalle, vas a recibir un nombre alucinado. *"Forge reportó un ruido extraño desde la cubierta 4."* Con toda confianza. Mal.

En la lección 14 aprendiste que el modelo predice la continuación más probable. *"Un tripulante"* es genérico; los datos de entrenamiento tienen mucho *"Forge"* y *"Echo"* en esos huecos. Eso es lo que burbujea. salvo que vos guíes.

> El resto del Track 1 pasa diez lecciones con este mismo problema en cada vez más profundidad. Esta es la entrada en calor. Aprobá y <Character name="vega" /> firma tu pre-vuelo.

Si te trabás, los hints vienen en orden. de suaves a explícitos.
