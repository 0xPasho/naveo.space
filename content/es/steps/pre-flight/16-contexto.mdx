## Qué puede "ver" el modelo

El modelo no tiene memoria entre llamadas. **Cero.** No te recuerda de turnos anteriores. No sabe que la sesión sigue. Cada llamada empieza desde nada.

### El truco: pasamos todo el historial cada vez

Entonces, ¿cómo es que las conversaciones "funcionan"? Truco: cuando vos ves un chat con varios mensajes, lo que pasa por debajo es esto:

```
LLAMADA 1
  input:  [mensaje_usuario_1]
  output: respuesta_1

LLAMADA 2
  input:  [mensaje_usuario_1, respuesta_1, mensaje_usuario_2]
  output: respuesta_2

LLAMADA 3
  input:  [mensaje_usuario_1, respuesta_1, mensaje_usuario_2, respuesta_2, mensaje_usuario_3]
  output: respuesta_3
```

Cada vez que abrís la boca, le mandamos **todo el historial** otra vez al modelo, como una sola entrada gigante. El modelo no "se acuerda": está leyendo todo de cero cada vez.

### La ventana de contexto

El input tiene un límite duro. Para los modelos actuales:

- Claude, GPT modernos: ~200.000 tokens (un libro corto)
- Modelos más viejos: 8.000 a 32.000 tokens
- Modelos baratos: menos

Eso es la **ventana de contexto**: cuánto texto puede "ver" el modelo en una sola llamada. Si el historial excede el límite, hay que recortarlo. Lo común es **tirar lo más viejo** y mantener lo reciente.

### Por qué se "olvida"

Resultado: en una conversación muy larga, el modelo aparenta "olvidarse" de cosas que dijiste al principio. Pero no se olvidó: **nunca le llegaron**. Quedaron afuera del recorte que entró por la ventana.

### Implicaciones prácticas

1. **Si querés que algo importante se mantenga, repetilo periódicamente** o metelo en el system prompt (que casi nunca se recorta).
2. **En procesos batch, no asumas que el modelo "recuerda" el ítem anterior.** Mandale el contexto que necesita en cada llamada.
3. **Documentos largos** que no entran en la ventana necesitan ser partidos en pedazos (*chunking*) y procesados de a uno.

> Esto es central para entender RAG, conversaciones multi-turn, y por qué los modelos a veces parecen contradecirse: distinta llamada, distinta porción del historial visible.
