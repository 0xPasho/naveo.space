## Repetir lo mismo, muchas veces

A veces no tenés que escribir el mismo paso seis veces. Lo escribís **una sola vez**, y le decís a la nave: *hacelo para cada tripulante*.

Eso es un **loop**. En código se ve así:

```
PARA CADA tripulante EN crew:
  verificar oxígeno del traje
```

Ese pedazo de protocolo se "expande" en tiempo de ejecución. Si la crew son 4 personas, corre 4 veces. Si son 12, corre 12 veces. **Vos no contás.** Es el loop el que cuenta.

### Dos versiones del mismo trabajo

Sin loop:

```
Verificar oxígeno de Vega
Verificar oxígeno de Atlas
Verificar oxígeno de Echo
Verificar oxígeno de Forge
```

Con loop:

```
PARA CADA tripulante EN crew:
  Verificar oxígeno del traje
```

La segunda no es solo más corta. Es **correcta cuando la crew cambia**. Si mañana se suma alguien, el loop la incluye automáticamente. La versión sin loop ignora a los nuevos hasta que te acuerdes de agregar la línea.

### Cómo reconocer un loop

Fijate si la acción tiene un sujeto que **cambia entre repeticiones**:

- *Verificar oxígeno del traje*: el traje es de uno distinto cada vez. **Loop.**
- *Encender los motores principales*: los motores son los mismos para todos. **Una sola vez.**

Si el sujeto siempre es el mismo, no necesitás loop. Si el sujeto va variando entre las corridas, sí.

> Cuando le mandes un prompt al modelo con una lista adentro (por ejemplo *"para cada bio, extraé el email"*), estás usando exactamente esta forma. El loop está adentro del prompt y se ejecuta una vez por bio.
