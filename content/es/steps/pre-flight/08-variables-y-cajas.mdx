## Una variable es una caja etiquetada

Imaginate cinco cajas en el puente. Cada una tiene una etiqueta pintada: `Nivel de combustible`, `Reserva de oxígeno`, `Rumbo`. La etiqueta nunca cambia. **Lo que cambia es el valor adentro.**

Eso es todo lo que es una variable. La caja (el nombre) es fija. El contenido (el valor) se puede cambiar, actualizar, leer, reemplazar.

```
Nivel de combustible = 87%
Reserva de oxígeno = 14.2 horas
Rumbo = 042° NO
```

La crew lee las etiquetas para saber qué pasa. <Character name="echo" /> lee `Reserva de oxígeno` para saber cuánto falta hasta que falle el soporte vital. <Character name="forge" /> lee `Nivel de combustible` para saber si los motores están sanos.

### Una caja, muchos valores en el tiempo

Mirá la caja `combustible` durante el primer cuarto de turno:

```
00:00  combustible = 100%
00:15  combustible = 92%
00:30  combustible = 85%
00:45  combustible = 78%
```

Cuatro snapshots distintos. Misma caja. **La etiqueta nunca se movió**. siempre se llamó `combustible`. Lo que se actualizó fue el valor adentro, porque la nave consumió combustible entre cada lectura.

Cuando <Character name="echo" /> te avise *"combustible bajo"*, está leyendo el valor **actual** de esa caja. No el de hace 15 minutos. La variable siempre se lee en el momento en que se la pide.

> Cuando escribís un prompt que dice `<bio>{{input}}</bio>`, estás haciendo lo mismo. `input` es una variable. La etiqueta se queda. El bio adentro cambia por cada llamada. exactamente como la caja `combustible` cambia entre lectura y lectura.

A la derecha: cinco indicadores etiquetados y cinco valores. Arrastrá cada valor al indicador que le va. Las etiquetas te dicen qué tipo de valor corresponde.
