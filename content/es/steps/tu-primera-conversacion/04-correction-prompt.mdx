## Editar, no regenerar

Una vez que el modelo te da un output **casi** bueno, la tentación es decir "reescribilo mejor". Mal movimiento. Cada palabra vaga ("más claro", "más profesional", "más conciso") es una licencia para reescribir todo, incluyendo las partes que ya estaban bien.

El correction prompt quirúrgico funciona al revés: nombrás los cambios y nombrás lo que **no** se toca.

### El patrón de 3 líneas

```
Misma respuesta pero con estos cambios:
1. <cambio específico>
2. <cambio específico>
No toques el resto.
```

Tres líneas. Lista numerada para que el modelo no se pierda. Cierre explícito de "no toques el resto" para evitar la reescritura silenciosa.

### Lo que NO funciona

- "Reescribilo, está medio raro." Le diste licencia total. Va a cambiar todo.
- "Más profesional." Tres palabras vagas, tres puertas abiertas. Va a meter formalismos donde no querías.
- "Mejorá esto." El modelo elige qué "mejorar". Spoiler: no es lo que vos pensabas.

### Lo que sí

- "Sacá el saludo y cambiá 'ayer' por 'el turno pasado'. Resto igual."

Dos cambios nombrados. Frase final de protección. El output viene con esos dos cambios y nada más.

> Esta técnica también te sirve cuando le pasás un código al modelo y querés que arregle UN bug sin tocar el resto. *"Cambiá solo la línea 12 para que devuelva null si el input es vacío. No reformatees el resto."* Si no lo decís, te devuelve el archivo entero con cinco cambios estéticos que vos no pediste.

A la derecha, el mismo follow-up en dos formas. ¿Cuál preserva el resto del output?
